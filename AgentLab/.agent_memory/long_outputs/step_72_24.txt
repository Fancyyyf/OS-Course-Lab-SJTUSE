Call `read_diff` to retrieve the pending patch diff.\n2. If necessary, call `read_patch_file` to review context files.\n3. Analyze the patch for security vulnerabilities, especially verifying path handling:\n   - Check if files are read/written using raw/user-controlled paths without safety limits.\n   - Look for path traversal risks where paths can escape workspace boundaries.\n4. If path handling is unsafe:\n   - Select `\"request_changes\"` as the verdict.\n   - Formulate comments that identify the `\"path traversal\"` risk.\n   - Recommend resolving paths using `\"workspace.resolve\"`.\n   - Request adding a regression `\"test\"`.\n5. Call `submit_review` to record the verdict and comments.\n\n## Checklist\n- **Verdict**: Must be `\"request_changes\"` if path traversal is found.\n- **Vulnerability**: Comments must explicitly mention `\"path traversal\"` or `\"workspace\"` boundaries.\n- **Remediation**: Comments must explicitly recommend `\"workspace.resolve\"`.\n- **Testing**: Comments must explicitly request a regression `\"test\"`.\n"}
