{"ok":true,"content":"## When To Use\nUse this skill when you need to redact sensitive data from a support ticket and submit the sanitized version.\n\n## Procedure\n1. Call `read_ticket` with the target `ticket_id` to retrieve the ticket's raw content.\n2. Identify and replace all sensitive data fields with placeholders (e.g., `[EMAIL]`, `[PHONE]`, `[STUDENT_ID]`, `[TOKEN]`, `[IP]`).\n3. Retain the operational support context (e.g., \"password reset\" and \"MFA enrollment\").\n4. Call `validate_redaction` with the draft redacted content.\n5. Fix any identified issues and re-run validation until no issues remain.\n6. Call `submit_redacted_ticket` to finalize the submission.\n\n## Checklist\n- **Email**: Replaced with a placeholder.\n- **Phone**: Replaced with a placeholder.\n- **Student ID**: Replaced with a placeholder.\n- **Access Token**: Replaced with a placeholder.\n- **Internal IP**: Replaced with a placeholder.\n- **Keywords**: Exact phrases `\"password reset\"` and `\"MFA enrollment\"` must be preserved.\n- **Validation**: Call `validate_redaction` before calling `submit_redacted_ticket`."}
